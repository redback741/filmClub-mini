## 1. 项目概览

- 技术栈：Taro 4.1.2 + React 18 + Sass（Vite 编译）

## 6. 数据层：请求封装、持久化、状态管理

### 6.1 请求封装（统一入口）

文件：[request.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/utils/request.js)

- Token 注入：从 `userStorage.getToken()` 读取，存在则追加 `Authorization: Bearer ${token}`
- 参数规则：
  - `GET/DELETE`：将 `params` 过滤后拼到 query string
  - `POST/PUT/PATCH`：将 `params` 放到 `data`（请求体）
- 统一返回结构（核心字段）：
  - 成功：`{ success: true, data, message, code }`
  - 失败：`{ success: false, data: null, message, code }`
  - 兼容后端 `data.code` 取值 `200/0/201` 视为成功

### 6.2 本地持久化（Storage）

文件：[storage.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/utils/storage.js)

- Key：
  - `STORAGE_KEYS.TOKEN = "token"`
  - `STORAGE_KEYS.USER_INFO = "userInfo"`
- 用户会话相关 API：
  - `userStorage.getToken/setToken/removeToken`
  - `userStorage.getUserInfo/setUserInfo/removeUserInfo`
  - `userStorage.clearUserSession()`：同时清理 token + userInfo

### 6.3 全局状态（类 Vuex：Context + reducer）

文件：[store/index.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/store/index.js)

- State：
  - `userInfo`：初始化时从 `userStorage.getUserInfo()` 水合
- Provider：
  - `StoreProvider`：在 [app.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/app.js#L7-L17) 挂载，包裹所有页面
- Hook：
  - `useStore()`：返回 `{ state, dispatch }`
  - `useUserInfo()`：返回 `state.userInfo`
- Action Creator：
  - `setUserInfo(userInfo)` -> `{ type: 'user/setUserInfo', payload }`
  - `clearUserInfo()` -> `{ type: 'user/clearUserInfo' }`
- 副作用：
  - reducer 内会同步写入/清除 `userStorage`（实现持久化一致性）

补充说明：

- [store/slices/userSlice.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/store/slices/userSlice.js) 属于历史 Redux Toolkit 方案的残留文件，目前项目入口未见 Redux store 挂载。

## 7. 接口封装（API）

接口封装统一使用 [request.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/utils/request.js)。

### 7.1 用户相关

文件：[user.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/api/user.js)

- `wxLogin(params)`：`POST /user/wx-login`  
  - 入参：`{ code }`（来自 `Taro.login()`）
- `updateUserInfo(params)`：`POST /user/update`
- `getUserInfo()`：`GET /user/info`
- `deleteUser()`：`DELETE /user/delete`

### 7.2 电影相关

文件：[movies.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/api/movies.js)

- `listMovies(params={})`：`GET /movie/list`
  - 页面使用示例：`listMovies({ isRecently: true })`

### 7.3 活动相关

文件：[activity.js](file:///Users/wty/Desktop/demo/filmClub-mini/src/api/activity.js)

- `listActivities(params={})`：`GET /activity/list`
  - 常用入参：`{ city, startTime }`
- `getActivityDetail(id)`：优先 `GET /activity/detail?id=...`；失败回退 `listActivities()` 后本地筛选
- `registerActivity(id)`：`POST /activity/register`（query：`id`）
- `registerListActivity()`：`GET /activity/register-list`
- `deleteActivity(id)`：`DELETE /activity/delete`（query：`id`）
- `updateActivity(id, params)`：`POST /activity/update?id=...`（body：`params`）
- `updateActivityStatus(id, status)`：`POST /activity/update-status?id=...&status=...`
- `listActivitiesByMovie(movieId, params={})`：`GET /activity/list-by-movie`（query：`movieId` + 额外参数）

### 8.7 我的页：登录与用户信息（Mine）

- 文件：[index.jsx](file:///Users/wty/Desktop/demo/filmClub-mini/src/pages/mine/index.jsx)
- 页面结构：
  - 用户信息卡片（点击触发登录）
  - “我的报名”入口
  - “退出登录”按钮（登录后显示）
- 变量内容（全局状态）：
  - `userInfo`：来自 `useUserInfo()`（Store Context）
  - `dispatch`：来自 `useStore()`
- 登录流程：
  - `Taro.login()` 获取 `code`
  - `wxLogin({ code })` 换取后端登录数据（并在成功时写入状态与 token）
  - `dispatch(setUserInfo(loginRes.data))`
  - `userStorage.setToken(loginRes.data.token)`
- 退出流程：
  - `dispatch(clearUserInfo())`（会清除 userInfo 的持久化）
- 调用接口：
  - `POST /user/wx-login`
